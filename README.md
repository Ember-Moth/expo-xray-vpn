# expo-xray-vpn

Expo native module for Android VPN — VpnService with Xray-core engine.

> ⚠️ Android only. Does not work in Expo Go.

## Installation

```bash
npm install expo-xray-vpn
```

## Requirements (consumer project)

- Expo SDK 55+
- Android Gradle project (generated by `expo prebuild`)

### Maven repository for AAR

Add GitHub Packages Maven repository to the consumer's root `android/build.gradle`:

```groovy
allprojects {
  repositories {
    google()
    mavenCentral()
    maven { url 'https://maven.pkg.github.com/Ember-Moth/expo-xray-vpn' }
  }
}
```

Authentication (required for GitHub Packages):

Create or edit `~/.gradle/gradle.properties`:

```properties
gpr.user=Ember-Moth
gpr.key=ghp_YOUR_GITHUB_TOKEN_WITH_PACKAGES_READ
```

### Or use local AAR (no Maven needed)

Place `libXray.aar` (built from [`Ember-Moth/libXray`](https://github.com/Ember-Moth/libXray)) in `node_modules/expo-xray-vpn/android/libs/libXray.aar`.
The module falls back to local AAR when Maven is unavailable.

## Usage

```typescript
import StarlinkVpn from 'expo-xray-vpn';

const permission = await StarlinkVpn.requestPermission();
if (!permission.granted) return;

const state = await StarlinkVpn.connect({
  profileName: 'My Node',
  xrayConfigJson: '...',
});
```

Full API reference: [USAGE.md](./USAGE.md)

## Module structure

```
expo-xray-vpn/
├── index.ts                    # TypeScript entry point
├── expo-module.config.json     # Expo autolinking config
├── src/                        # TypeScript source
│   ├── StarlinkVpnModule.ts
│   └── StarlinkVpn.types.ts
├── android/
│   ├── build.gradle            # Android library (expo-module-gradle-plugin)
│   └── src/main/
│       ├── AndroidManifest.xml
│       ├── assets/xray/        # geoip.dat / geosite.dat (auto-downloaded)
│       └── java/.../vpn/       # Kotlin sources
└── libXray/                    # Git submodule for AAR source
    └── publish/                # Gradle Maven publish config
```

## AAR (libXray) dependency

The native Xray-core engine is distributed as an Android AAR:

| Source | How | When |
|--------|-----|------|
| **GitHub Packages Maven** | `com.starlink.vpn:libxray:VERSION@aar` | CI / production |
| **Local file** | `android/libs/libXray.aar` | Local development |
| **Build from source** | `cd libXray && python3 build/main.py android && cp libXray.aar ../android/libs/` | Custom builds |

## Publishing

### npm package

Trigger the [`publish` workflow](.github/workflows/publish.yml) via GitHub Actions:

- **Manual:** `workflow_dispatch` with version bump type
- **Auto:** push tag `v*` (e.g. `v0.8.0`)

### AAR to Maven (manual)

```bash
cd libXray/publish
export LIBXRAY_VERSION=1.260509.0-patch
./gradlew publishAarPublicationToGitHubPackagesRepository
```
